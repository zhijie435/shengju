# 环境配置说明

## 配置文件位置

环境配置文件位于 `backend/.env`，该文件不会被提交到版本控制系统。

## 配置项说明

### 数据库配置

- `DB_HOST`: 数据库主机地址（默认: localhost）
- `DB_PORT`: 数据库端口（默认: 3306）
- `DB_USER`: 数据库用户名（默认: root）
- `DB_PASSWORD`: 数据库密码（默认: 空）
- `MAIN_DB_NAME`: 主数据库名称（默认: question_management_shared）

### 安全配置

- `JWT_SECRET`: JWT令牌密钥（**生产环境必须修改为强密码**）

### 服务器配置

- `PORT`: 服务器监听端口（默认: 3000）。**若 3000 被占用**，可改为 3001、3002 等，并参见下文「更换后端端口」。
- `NODE_ENV`: 运行环境（development/production）

### 文件上传配置

- `UPLOAD_DIR`: 文件上传目录（默认: ./uploads）
- `MAX_FILE_SIZE`: 最大文件大小（默认: 50mb）

## 初始化配置

### 方法一：使用交互式脚本（推荐）

运行以下命令：

```bash
node backend/scripts/init-config.js
```

或在 Windows 上运行：

```bash
初始化环境配置.bat
```

### 方法二：快速初始化

运行以下命令：

```bash
快速初始化环境.bat
```

这会从 `.env.example` 复制模板创建 `.env` 文件，然后你需要手动编辑配置。

### 方法三：手动创建

1. 复制 `backend/.env.example` 为 `backend/.env`
2. 编辑 `backend/.env` 文件，修改相应的配置值

## 配置示例

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
MAIN_DB_NAME=question_management_shared

# JWT密钥（生产环境请务必修改为强密码）
JWT_SECRET=your-very-secure-secret-key-here

# 服务器配置
PORT=3000
NODE_ENV=development

# 文件上传配置
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=50mb
```

## 更换后端端口（端口 3000 被占用时）

1. 在 `backend/.env` 中设置新端口，例如：`PORT=3001`
2. 在每个要访问该后端的**前端项目**下创建或编辑 `.env`，增加一行（端口与上一步一致）：
   ```env
   VITE_API_PORT=3001
   ```
   涉及项目：`frontend/exam-admin`、`frontend/exam-super-admin`、`frontend/exam-student`、`frontend/exam-grader`
3. 重启后端服务，并重启已打开的前端开发服务

## 注意事项

1. **生产环境安全**：
   - 必须修改 `JWT_SECRET` 为强密码
   - 不要将 `.env` 文件提交到版本控制系统
   - 使用强密码保护数据库

2. **数据库配置**：
   - 确保 MySQL 服务已启动
   - 确保数据库用户有足够的权限
   - 确保数据库已创建（运行 `初始化数据库.bat`）

3. **端口冲突**：
   - 如果端口 3000 被占用，修改 `PORT` 配置
   - 修改后需要重启服务器

## 验证配置

启动服务器后，检查控制台输出：

```
✓ 主数据库连接成功
  - 数据库: question_management_shared
  - 主机: localhost:3306
服务器运行在端口 3000
```

如果看到错误信息，请检查：
1. `.env` 文件是否存在
2. 配置值是否正确
3. MySQL 服务是否启动
4. 数据库是否已创建
