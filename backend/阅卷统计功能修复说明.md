# 阅卷统计功能修复说明

## 问题描述

前端显示"阅卷统计功能开发中"的提示，但实际上后端API已经实现，只是前端缺少对应的页面和API调用。

## 修复内容

### 1. 前端API接口（exam-admin 和 exam-super-admin）

**文件：** `frontend/exam-admin/src/api/grading.js` 和 `frontend/exam-super-admin/src/api/grading.js`

**添加的API方法：**
- `getExamSummary(sessionId)` - 获取单个考生的汇总数据
- `getExamSummaries(examId, params)` - 获取考试的所有考生汇总数据
- `generateExamSummary(sessionId)` - 手动生成汇总数据
- `getExamStatistics(examId)` - 获取考试整体统计信息
- `getEvaluationReport(sessionId)` - 获取评估报告
- `generateEvaluationReport(sessionId)` - 生成评估报告
- `getEvaluationReports(examId, params)` - 获取考试的所有评估报告
- `downloadEvaluationReport(sessionId)` - 下载评估报告

### 2. 阅卷统计页面组件

**文件：** 
- `frontend/exam-admin/src/views/GradingStatistics.vue`
- `frontend/exam-super-admin/src/views/GradingStatistics.vue`

**功能特性：**
- **整体统计**：显示参考人数、平均分、平均得分率、及格率、最高分、最低分
- **考生汇总列表**：显示所有考生的详细汇总数据，支持分页
- **考生详情**：查看单个考生的详细统计，包括：
  - 各题型统计
  - 各难度统计
  - 各考察目的统计
- **评估报告**：查看和下载AI生成的评估报告

### 3. 路由配置

**文件：**
- `frontend/exam-admin/src/router/index.js`
- `frontend/exam-super-admin/src/router/index.js`

**添加的路由：**
```javascript
{ 
  path: 'grading-system/statistics/:examId', 
  name: 'GradingStatistics', 
  component: () => import('../views/GradingStatistics.vue') 
}
```

### 4. 更新GradingSystem组件

**文件：**
- `frontend/exam-admin/src/views/GradingSystem.vue`
- `frontend/exam-super-admin/src/views/GradingSystem.vue`

**修改：**
- 将 `viewStatistics` 函数从显示"开发中"提示改为跳转到统计页面

### 5. 后端修复

**文件：** `backend/routes/grading.js`

**修复：**
- 在 `checkAndGenerateSummaryAndReport` 函数中，查询答案时添加了 `exam_purpose` 和 `difficulty` 字段，确保统计功能能正确获取这些数据

## 使用说明

### 访问统计页面

1. 在阅卷系统页面，点击考试列表中的"阅卷统计"按钮
2. 系统会跳转到统计页面，显示该考试的整体统计和考生汇总

### 查看考生详情

1. 在考生汇总列表中，点击"查看详情"按钮
2. 弹出对话框显示该考生的详细统计信息，包括：
   - 基础信息（总分、得分率、答题时长等）
   - 各题型统计
   - 各难度统计
   - 各考察目的统计

### 查看评估报告

1. 在考生汇总列表中，点击"查看报告"按钮
2. 如果报告已生成，会显示完整的评估报告
3. 如果报告未生成，可以点击"生成报告"按钮手动生成
4. 报告生成完成后，可以点击"下载报告"按钮下载HTML格式的报告

## 注意事项

1. **汇总数据自动生成**：当所有题目阅卷完成后，系统会自动生成汇总数据
2. **评估报告生成**：评估报告需要配置DeepSeek API密钥才能生成
3. **考察目的字段**：建议在创建试卷时为每道小题设置"考察目的"，这样统计和报告会更加详细
4. **权限控制**：只有管理员和企业用户可以查看统计信息

## 测试建议

1. 完成一次考试的阅卷
2. 点击"阅卷统计"按钮，查看整体统计是否正确
3. 查看考生汇总列表，确认数据正确
4. 点击"查看详情"，验证详细统计信息
5. 点击"查看报告"，测试评估报告功能（需要配置AI密钥）
