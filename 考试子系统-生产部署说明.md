# 考试子系统生产部署（企业端 / 考生端 / 子阅卷 / 总管理）

公网 IP 以下用 **`YOUR_IP`** 代替（如 `39.105.98.161`）。后端端口以 **`PORT=3002`** 为例。

## 一、一次性打包（在项目根目录 `新的/`）

```bash
cd /var/www/新的
```

各子项目需能读到 **`VITE_API_PORT`**（与 `backend/.env` 的 `PORT` 一致）。可在**每个** `frontend/exam-*` 目录下放 `.env`：

```env
VITE_API_PORT=3002
```

然后执行（四端一起打）：

```bash
npm run build:all-exam-frontends
```

或分别执行：

```bash
npm run build:exam
npm run build:exam-admin
npm run build:exam-student
npm run build:grader
```

完成后应存在目录：

- `frontend/exam-super-admin/dist`
- `frontend/exam-admin/dist`
- `frontend/exam-student/dist`
- `frontend/exam-grader/dist`

### 打包后务必核对（避免白屏、控制台 `/assets` 404）

打开 **`frontend/exam-student/dist/index.html`**（其余子系统同理），其中 `script`、`link` 的地址**必须**带各自前缀，例如考生端：

- ✅ 正确：`src="/exam-student/assets/index-xxxx.js"`
- ❌ 错误：`src="/assets/index-xxxx.js"`（浏览器会去 `http://你的IP/assets/...`，根路径下没有这些文件 → **404 + 白屏**）

若仍是 `/assets/`，说明当时**不是**用生产模式打包（例如误用 `vite build --mode development`），或上传的是旧目录。请在对应目录执行 **`npm run build`** 后**整目录覆盖**服务器上的 `dist`，再 **Ctrl+F5** 强刷。

在 Linux 服务器上可快速检查：

```bash
grep -E 'src=|href=' /var/www/新的/frontend/exam-student/dist/index.html
```

应能看到 `/exam-student/assets/`。

## 三、经 Nginx（80 端口，推荐）

在 **`server { listen 80; ... }` 里**、**`location / { proxy_pass ... }` 之前**，加入（路径按服务器实际调整）：

```nginx
    location /exam-super-admin/ {
        alias /var/www/新的/frontend/exam-super-admin/dist/;
        index index.html;
        try_files $uri $uri/ /exam-super-admin/index.html;
    }
    location /exam-admin/ {
        alias /var/www/新的/frontend/exam-admin/dist/;
        index index.html;
        try_files $uri $uri/ /exam-admin/index.html;
    }
    location /exam-student/ {
        alias /var/www/新的/frontend/exam-student/dist/;
        index index.html;
        try_files $uri $uri/ /exam-student/index.html;
    }
    location /exam-grader/ {
        alias /var/www/新的/frontend/exam-grader/dist/;
        index index.html;
        try_files $uri $uri/ /exam-grader/index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:3002;
        proxy_set_header Host $host;
    }

    location / {
        proxy_pass http://127.0.0.1:3002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
```

若你**不想**拆 `location /api/`，也可维持**仅** `location /` 反代整站；此时静态仍可由 **Node 的 `mountExamSpaSubapp`** 提供（只要请求能到 3002）。拆出 `/api/` 可减轻静态与 API 混用时的歧义。

执行：

```bash
nginx -t && systemctl reload nginx
```

## 六、`try_files` 与 SPA 若 404

部分环境下 `alias` + `try_files` 最后一项需用命名 location，可改为（示例仅 `exam-admin`）：

```nginx
    location /exam-admin/ {
        alias /var/www/新的/frontend/exam-admin/dist/;
        try_files $uri $uri/ @exam_admin_spa;
    }
    location @exam_admin_spa {
        rewrite ^ /exam-admin/index.html break;
        root /var/www/新的/frontend/exam-admin/dist;
    }
```

其余三个子系统同理替换前缀与路径。
