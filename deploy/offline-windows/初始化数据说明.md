# 初始化数据说明

## 一、数据来源

安装后**首次启动**时，系统会根据数据库真实状态自动导入初始化数据：

- **初始化文件**：`app\backend\database\seeds\qms_production_backup.sql`
- **目标数据库**：`question_management_shared`
- **原始文件**：`backend\database\seeds\qms_full_backup.sql`（MariaDB 10.5.29 dump，约 1.27 MB）
- **包含内容**：用户、企业、题库、试卷、考试、报名等完整生产数据

> 说明：尝试通过 SSH 连接生产服务器导出最新数据时，给定地址 `105.98.161`
> 为不完整的 IPv4（仅 3 段，无法解析），且补全为 `105.98.161.1~254`、`105.98.0.161`
> 等常见格式后全部连接超时，服务器当前不可达。因此使用项目内已有的
> `qms_full_backup.sql` 作为初始化数据源。若需替换为最新生产数据，直接覆盖
> `deploy/offline-windows/packaging/app/backend/database/seeds/qms_production_backup.sql` 后重新打包即可。

## 二、首次启动初始化流程（数据驱动判断）

`启动考试系统.bat` 的 `[4/5]` 步骤不再依赖"数据目录是否存在"，而是**查询真实表数据**决定是否导入：

1. 启动 MariaDB，创建 `question_management_shared` 库
2. 查询 `question_management_shared.users` 表的行数（表不存在则视为 0）
3. **行数为 0 且存在 SQL 文件** → 导入 `qms_production_backup.sql`，成功后写入 `.seed_done` 标记
4. 导入失败 → 回退执行 `seed_initial_data.js` 默认种子脚本
5. **行数大于 0** → 跳过导入（避免覆盖已有数据）
6. 导入日志：`logs\sql_import.log`，用户数探测：`logs\user_count.txt`

## 四、重置数据库

运行桌面/开始菜单的 **重置数据库**，或安装目录下的 `重置数据库.bat`。重置会删除：

- MariaDB 数据目录（`data\`）
- 种子标记文件（`data\.seed_done`）

重置后下次启动将重新从 SQL 文件导入全部生产数据。

## 五、bat 文件编码说明（乱码修复）

所有 bat 文件统一为 **GBK/ANSI 编码 + `chcp 936`**（中文 Windows cmd.exe 原生组合），
从 git 干净 UTF-8 源**单次转换**生成，已做字节级验证（中文 lead byte 在 0x81-0xFE，
无 `?` 替换字符、无 U+FFFD）。打包流程（`build-local.ps1` Step 8a）由
`patch_and_convert.ps1`（UTF-8 with BOM，幂等）统一完成：git 恢复 → 注入逻辑 → 转 GBK → 验证。
