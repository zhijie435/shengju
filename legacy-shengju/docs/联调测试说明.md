# 联调测试说明

本文档说明管理端、求职者端、企业端与后端联调时的关键验证点。

---

## 1. 管理端

### 1.1 侧边栏与跳转

- **圣举测评系统**：侧边栏显示「圣举测评系统」，点击应在**新标签页**打开当前主机 3000 端口（如 `http://localhost:3000/src/app.html#examSystem` 或通过其他电脑访问时 `http://<服务器IP>:3000/src/app.html#examSystem`），跳转正确。
- **政企对话**：侧边栏显示「政企对话」，点击应跳转到 `custom-solutions.html#enterprise-chat`，定位到管理者-政企端对话入口。

以上入口在 `admin/report-templates.html`、`admin/index.html` 等管理页侧边栏中均已配置，请逐页验证点击无误。

---

## 2. 会员购买与通知

- 在**求职者端**购买一次会员（如从 `user/payment.html` 完成下单）。
- 在**企业端**购买一次会员（企业端会员购买流程完成一次）。
- 在**管理端首页**（`admin/index.html`）：
  - 顶部铃铛中应出现对应**通知**（如会员订单/企业认证等）。
  - **角标**数量与未读条数一致。
  - 可**单条标记已读**、**全部已读**，操作后角标与列表状态正确更新。

通知接口：`GET /api/v1/admin/notifications`、`PUT .../read`、`PUT .../read-all`，需登录管理员账号。

---

## 3. 系统设置与三端展示

- 在**管理端**进入「系统设置」（`admin/settings.html`），修改并保存：
  - **站点名称**（siteName）
  - **站点 Logo**（siteLogo）
  - **联系方式**（contactInfo：电话、邮箱、地址等）
- 保存后：
  - **求职者端**：刷新 `user/index.html` 等页，导航栏 Logo、站点名与页脚联系方式与设置一致。
  - **企业端**：刷新 `enterprise/index.html`、`enterprise/company-center.html`，导航栏与页脚与设置一致。
- **再次刷新浏览器**（或重新打开页面）：上述展示仍与设置一致，说明数据来自**数据库**（`sj_system_settings`），非仅前端缓存。

前台通过**公开接口** `GET /api/v1/admin/settings/public` 拉取 siteName、siteLogo、contactInfo，无需登录；拉取后可写入 localStorage（如 siteName、siteLogo、contactInfo、systemSettings）供离线或同源页使用。

---

## 4. 报告模板与简历模板

### 4.1 报告模板

- 报告模板当前为**前端维护**（如 localStorage 的 `reportTemplates` 及管理端报告模板列表页的默认数据），**无独立报告模板表**。
- 若存在仅用于测试的条目（名称含「测试」的占位数据），应在数据源中删除或禁用，并确保列表接口/前端只返回正式数据。
- 当前默认报告模板列表为正式业务模板，无测试占位数据。

### 4.2 简历模板（管理端与求职者端一致）

- **管理端**「报告模板管理」页中的「简历管理」选项卡：列表数据来自 **GET /api/v1/resume-templates**，与**求职者端个人中心**「我的简历」所用模板列表为**同一数据源**。
- 管理端对简历模板的**编辑、禁用/删除**通过 **admin 简历模板 API** 完成：
  - `PUT /api/v1/admin/resume-templates/:id`：更新名称、内容、分类、状态等
  - `DELETE /api/v1/admin/resume-templates/:id`：禁用该模板（status=0），禁用后求职者端列表不再展示
- 联调时验证：在管理端修改或禁用某简历模板后，刷新求职者端个人中心，模板列表与管理端一致且无测试数据。

---

## 5. 后端与数据

- **公开设置接口**：`GET /api/v1/admin/settings/public` 返回 `{ success, data: { siteName, siteLogo, contactInfo } }`，无需 Authorization。
- **简历模板**：表 `sj_resume_templates`，公开接口只返回 `status=1` 的模板；管理端通过 `/api/v1/admin/resume-templates` 做列表（含禁用）/ 单条 / 创建 / 更新 / 禁用。
- 修改后台代码或配置后，需**重启后端服务**（如 `node backend/src/index.js` 或项目既定启动方式）使接口生效。
