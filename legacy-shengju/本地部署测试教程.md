# 圣举人才网 + 考试系统 本地部署测试教程

本文档说明如何在本地同时部署 **圣举人才网**（求职者/企业/管理端门户）与 **考试系统**（线上笔试、阅卷、题库），用于联调与功能测试。

---

## 一、环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| **Node.js** | 16 及以上 | 运行前后端，[官网下载](https://nodejs.org/) |
| **npm** | 随 Node 安装 | 包管理 |
| **MySQL** | 5.7 / 8.0 | 两个系统共用同一 MySQL 实例，使用不同数据库 |

检查安装：

```bash
node -v   # 应显示 v16.x 或更高
npm -v
mysql --version
```

### 1.2 目录结构说明

本地测试会涉及两个工程（可放在不同文件夹）：

- **圣举人才网（本目录）**  
  - 后端：`backend/`（端口 **3001**，数据库 **shengju**）  
  - 前端：静态 HTML（`index.html`、`admin/`、`user/`、`enterprise/`），由后端一起提供  

- **考试系统**（在「圣举人才网\新的」目录下）  
  - 后端：`backend/`（端口 **3000**，数据库 **question_management_shared**）  
  - 前端：4 个 Vite 项目（企业端、考生端、总管理端、阅卷端），各自不同端口  

### 1.3 运行环境软件都安装在哪里？能都移到与系统一个盘吗？

下面说明各软件默认安装位置，以及如何**统一放到与系统同一盘（如 C:）或同一目录**，方便管理。

| 软件/内容 | 默认安装位置（Windows） | 说明 |
|-----------|-------------------------|------|
| **Node.js** | `C:\Program Files\nodejs\` | 安装时可选“自定义目录”，可改为如 `C:\Dev\nodejs\` |
| **npm** | 随 Node.js 安装在同一目录 | 无需单独安装 |
| **npm 全局包** | `C:\Users\你的用户名\AppData\Roaming\npm` | 若改过 Node 安装路径，一般也在此用户目录下 |
| **MySQL** | `C:\Program Files\MySQL\MySQL Server 8.0\`（或类似） | 安装时可选择目录，如 `C:\Dev\MySQL\` |
| **项目依赖 node_modules** | 各项目下的 `backend/node_modules`、`frontend/xxx/node_modules` | 每个项目一份，不能多个项目共用一个文件夹 |

**能否都移到与系统一个盘/同一个父文件夹？**

- **可以统一到与系统一个盘（如 C:）**  
  - 安装 Node.js、MySQL 时选择**自定义路径**，例如都放在 `C:\Dev\` 下：  
    - Node.js：`C:\Dev\nodejs\`  
    - MySQL：`C:\Dev\MySQL\`  
  - 这样“运行环境”都在 C 盘同一父目录，便于备份或重装系统后恢复。

- **推荐做法（与系统同一盘 + 项目也集中）**  
  1. **运行环境**：安装时统一选同一盘、同一父目录，例如：  
     - `C:\Dev\nodejs\`  
     - `C:\Dev\MySQL\`  
  2. **项目代码**：把圣举人才网、考试系统两个工程都放在同一盘下同一个父文件夹，例如：  
     - `C:\Dev\shengju\`（圣举人才网，即本「新建文件夹」内容）  
     - `C:\Dev\shengju-exam\`（考试系统，即「圣举人才网\新的」内容）  
  3. **node_modules**：保持在各项目内的 `backend`、`frontend/xxx` 下，不要手动移动；用 `npm install` 安装即可，路径会随项目走。

- **不建议**  
  - 把多个项目的 `node_modules` 合并到一个文件夹（会破坏依赖）。  
  - 安装好后用“剪切/粘贴”把 Node 或 MySQL 目录挪到别处（会导致命令、服务失效），应使用**重新安装并选择目标路径**的方式。

**查看当前安装位置（本机）：**

```bash
# Node.js 安装目录
where node

# npm 全局目录
npm config get prefix
```

MySQL 安装目录可在“服务”里查看该服务对应可执行文件路径，或在安装时记下所选路径。

## 二、数据库准备（MySQL）

两个系统使用 **同一台 MySQL 实例**，但使用 **两个数据库**，互不覆盖。

### 2.1 启动 MySQL 服务

- Windows：服务管理器或 `net start MySQL80`（名称以本机为准）  
- macOS：`brew services start mysql` 或 系统偏好设置  
- Linux：`sudo systemctl start mysql`  

### 2.2 创建数据库（仅建库，表由后续脚本创建）

用 MySQL 客户端或 Navicat 等执行：

```sql
-- 圣举人才网主站
CREATE DATABASE IF NOT EXISTS shengju DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 考试系统（题库、阅卷、线上笔试）
CREATE DATABASE IF NOT EXISTS question_management_shared DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.3 圣举人才网数据库初始化（shengju）

在 **圣举人才网（本目录）** 下执行：

```bash
# 方式一：命令行导入（将 your_password 换成 root 密码）
mysql -u root -p shengju < backend/scripts/init-db.sql

# 方式二：在 MySQL 客户端里
USE shengju;
SOURCE 这里填写 backend/scripts/init-db.sql 的完整路径;
```

若有其他迁移或初始化脚本（如 `migrate-*.sql`），按项目说明依次执行。

### 2.4 考试系统数据库初始化（question_management_shared）

进入 **考试系统** 所在目录（即「圣举人才网\新的」）：

```bash
cd 圣举人才网\新的\backend
npm install
```

复制环境配置并编辑（见下一节），然后执行迁移与测试账号创建：

```bash
# 若项目提供试卷/在线考试迁移脚本，按顺序执行，例如：
node scripts/run_exam_papers_migration.js
node scripts/run_online_exam_migration.js
# 其他迁移按项目 README 或 docs 执行

# 创建测试账号（管理端、企业端、考生端）
node scripts/create_test_accounts.js
```

具体脚本名以项目内 `package.json` 的 scripts 和 `docs/` 为准。

## 三、圣举人才网（本目录）配置与启动

### 3.1 后端环境变量

```bash
cd backend
copy .env.example .env   # Windows
# 或 cp .env.example .env  # macOS/Linux
```

编辑 `backend/.env`，至少配置：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=shengju

JWT_SECRET=请改为随机长字符串
JWT_EXPIRES_IN=7d

PORT=3001
```

### 3.2 安装依赖并启动后端

```bash
cd backend
npm install
npm start
```

成功时控制台会看到类似：

```
圣举人才网 API 已启动: http://127.0.0.1:3001
前端可访问: http://127.0.0.1:3001/index.html
```

后端会同时提供：

- API：`http://127.0.0.1:3001/api/v1/...`
- 静态前端：`http://127.0.0.1:3001/index.html`（首页）、`/admin/`、`/user/`、`/enterprise/` 等

### 3.3 访问圣举人才网

在浏览器打开：

| 入口 | 地址 |
|------|------|
| 首页 | http://127.0.0.1:3001/index.html |
| 求职者端 | http://127.0.0.1:3001/user/index.html |
| 企业端 | http://127.0.0.1:3001/enterprise/index.html |
| 管理端 | http://127.0.0.1:3001/admin/index.html |

**注意**：前端页面中 API 默认指向 `http://127.0.0.1:3001/api/v1`，必须通过上述地址访问，不要用 `file://` 打开，否则接口会跨域或无法请求。

## 四、考试系统（圣举人才网\新的）配置与启动

### 4.1 后端环境变量

在 **圣举人才网\新的** 目录下：

```bash
cd backend
copy .env.example .env   # Windows
# 或 cp .env.example .env
```

编辑 `backend/.env`，至少配置：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
MAIN_DB_NAME=question_management_shared

JWT_SECRET=请改为另一组随机长字符串（可与人才网不同）

PORT=3000
NODE_ENV=development
```

若 3000 端口被占用，可改为 `PORT=3001` 等，并确保与下面前端代理端口一致（见 4.3）。

### 4.2 安装依赖并启动考试系统后端

```bash
cd backend
npm install
npm start
# 或 npm run dev
```

成功时会看到类似：

```
服务器运行在端口 3000
✓ 主数据库连接成功
```

### 4.3 启动考试系统前端（4 个开发服务器）

需要 **4 个终端**，分别启动 4 个前端（端口已在各项目 `vite.config.js` 中配置好）：

**终端 1 - 企业端（端口 5174）**

```bash
cd frontend/exam-admin
npm install
npm run dev
```

**终端 2 - 考生端（端口 5176）**

```bash
cd frontend/exam-student
npm install
npm run dev
```

**终端 3 - 总管理端（端口 5175）**

```bash
cd frontend/exam-super-admin
npm install
npm run dev
```

**终端 4 - 阅卷端（端口 5177）**

```bash
cd frontend/exam-grader
npm install
npm run dev
```

若后端未用 3000（例如改为 3001），需在以上各前端项目根目录创建或修改 `.env`，增加一行：

```env
VITE_API_PORT=3001
```

然后重启对应前端的 `npm run dev`。

### 4.4 考试系统访问地址与测试账号

| 端 | 地址 | 测试账号（执行 create_test_accounts.js 后） |
|----|------|----------------------------------------------|
| 企业端 | http://localhost:5174 | enterprise / 123456（或文档中说明的密码） |
| 考生端 | http://localhost:5176 | candidate / 123456 |
| 总管理端 | http://localhost:5175 | admin / admin123 |
| 阅卷端 | http://localhost:5177 | 需在管理端/企业端创建阅卷账号并分配任务 |

具体账号密码以 `README_TEST_ACCOUNTS.md` 或脚本输出为准。

## 五、端口与关系汇总

| 服务 | 端口 | 说明 |
|------|------|------|
| 圣举人才网后端 | 3001 | API + 静态页面（index、admin、user、enterprise） |
| 考试系统后端 | 3000 | 题库、阅卷、线上笔试 API |
| 考试-企业端 | 5174 | Vite 开发服务器 |
| 考试-总管理端 | 5175 | Vite 开发服务器 |
| 考试-考生端 | 5176 | Vite 开发服务器 |
| 考试-阅卷端 | 5177 | Vite 开发服务器 |
| MySQL | 3306 | 默认 |

- 圣举人才网前端统一通过 **http://127.0.0.1:3001** 访问，无需再起其他静态服务。  
- 考试系统前端通过 Vite 各自端口访问，并透过 `VITE_API_PORT` 访问考试后端（默认 3000）。

## 六、一键启动建议（本地测试）

1. **先启动 MySQL**，并确认两个库 `shengju`、`question_management_shared` 已创建且已执行初始化/迁移脚本。  
2. **圣举人才网**：在一个终端 `cd backend && npm start`，浏览器访问 http://127.0.0.1:3001/index.html 。  
3. **考试系统**：在一个终端 `cd backend && npm start`，在另外 4 个终端分别启动 4 个前端的 `npm run dev`，再按上表地址访问。

可将上述命令写成批处理或 shell 脚本，便于本地一键启动（每个服务一个终端或后台进程）。

## 七、常见问题

**1. 后端启动报错 “数据库连接失败”**  
- 检查 MySQL 是否已启动。  
- 检查 `.env` 中 `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME` / `MAIN_DB_NAME` 是否正确。  
- 确认对应数据库已创建且已执行建表/迁移脚本。

**2. 人才网页面打开后接口 404 或跨域**  
- 必须通过 http://127.0.0.1:3001 访问页面，不要用 `file://`。  
- 确认人才网后端已启动且监听 3001。

**3. 考试系统前端报错 “Network Error” 或代理错误**  
- 确认考试系统后端已启动（默认 3000）。  
- 若后端改了端口，在前端项目根目录 `.env` 中设置 `VITE_API_PORT=实际端口` 并重启 `npm run dev`。

**4. 端口被占用**  
- 人才网后端：修改 `backend/.env` 中 `PORT=3001` 为其他未占用端口，前端页面仍通过该端口访问。  
- 考试后端：修改 `backend/.env` 中 `PORT`，并同步修改各前端 `.env` 的 `VITE_API_PORT`。  
- 前端端口：在对应 `frontend/*/vite.config.js` 的 `server.port` 中修改。

**5. 测试账号不存在或无法登录**  
- 考试系统：在「圣举人才网\新的\backend」下执行 `node scripts/create_test_accounts.js`。  
- 圣举人才网：若项目提供种子或注册接口，通过管理端或注册页创建账号，或查看项目内是否有 seed 脚本。

---

按以上步骤即可在本地完成圣举人才网与考试系统的部署与联调测试。若项目后续增加环境变量或迁移脚本，以项目内 `README`、`docs/` 和 `backend/.env.example` 为准。
