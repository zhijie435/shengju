## 为什么页面上看不到？

**子系统是独立前端**，和「考试系统」不是同一个项目：

- 根目录的 **`npm run build:exam`** 只构建「考试系统」（exam-super-admin），**不会**构建子系统。
- 你每天打开的「子系统」页面，来自**当时那一次**构建或开发服务器的结果。  
  若没有用当前源码重新构建或重新跑开发服务，浏览器里就一直是旧版。

## 怎么才能看到修改？

任选一种方式，让页面用上**当前源码**：

### 方式一：开发环境（改完就能看）

在**本目录**（`frontend/exam-grader`）执行：

```bash
npm run dev
```

然后用浏览器打开终端里提示的地址（默认 **http://localhost:5177**）。  
每次改完代码保存后，刷新页面即可看到最新效果。

### 方式二：生产构建（部署用）

在**项目根目录**执行：

```bash
npm run build:grader
```

或在**本目录**执行：

```bash
npm run build
```

构建完成后，会把结果输出到本目录的 **`dist`**。  
把你**实际访问子系统时用的那个网站/服务器**，指向并发布这个新的 `dist`，你每天用的那个入口就会变成带「子系统」和新评分表的版本。

---

**总结**：修改都在 `frontend/exam-grader/src/` 里；要看到效果，需要在本目录 `npm run dev` 或用当前代码重新 `npm run build` 并部署你平时用的那个地址。
